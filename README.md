
* TDD -> TFD + Refactoring

1. 1차적으로 도메인 테스트 연습 
2. todo list 작성(도메인 개념 습득)
3. tdd 로 구현하기 쉬운 코드부터 작성(예 : 유틸, input/output 정확한 것)
4. 큰 단위 -> 작은 단위로 쪼개기
5. 객체의 인스턴스 변수에 접근 x -> 메시지를 보내라
6. 객체끼리 비교(equals,hashcode)
7. 원시값을 객체로 포장해라
    - 유효청 체크 가능
    - 로직을 담을 수 있다
8. 일급 콜렉션을 쓴다
9. getter, setter -> 객체와 객체를 비교
